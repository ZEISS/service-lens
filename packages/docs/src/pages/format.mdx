# Specification

Lenses are defined in the JSON-format. These JSON files are used to share best-pratices and requirements between many workloads. The JSON-format is great to be stored in code repositories and shared via GitHub. Lenses contain four components.

* Pillars
* Questions
* Risks
* Improvments

The current version of the Lens Format Specification is the following.

```json
version: 1
```

## Lens section

This section defines the attributes for the lens itself. This is its name and a brief description.

| Field | Description |
| :--- | :----: |
| `version` | The version of the lens schema used (Default: `1`). |
| `name` | Name of the lens. THe name be up to 128 characters. |
| `description` | Text description of the lens. This text is displayed when selecting a lens for review. The description can be up to 2048 characters. |

## Pillars section

This section defines the pillars associated with the lens. These are the building block that contain different categories of questions and improvments.

| Field | Description |
| :--- | :----: |
| `id` | ID of the pillar. This needs to be unique between 3 and 128 characters and contain only alphanumeri and underscore (`_`) characters. |
| `name` | Name of the pillar. The name can be up to `128` characters.
| `description` | The description of the pillar. This text is displayed when evaluting the lens. The description can be up to 2048 characters. |

### Example

```JSON
"pillars": [
        {
            "id": "performance",
            "name": "Performance",
            "description": "..."
            .
            .
            .
        },
        {
            "id": "sustainability",
            "name": "Sustainability",
            "description": "..."
            .
            .
            .
        }
     ]
```

## Questions section

This section defines the questions contained in a pillar.

| Field | Description |
| :--- | :----: |
| `id` | ID of the question. This needs to be unique between 3 and 128 characters and contain only alphanumeri and underscore (`_`) characters. |
| `name` | Name of the question. The name can be up to `128` characters.
| `description` | The description of the question. This text is displayed when answering the question. The description can be up to 2048 characters. |
| `resources` | Contains a list of further resources to help understand the context of the question. |

### Example

```JSON
"questions": [
    {
        "id": "interface02",
        "name": "How do you monitor the interfaces with other systems?",
        "description": "We require that new interfaces monitor the inbound and outbound data to systems.",
        "resources": [
            {
                "description": "This is helpful text for the first question",
                "url": "https://example.com/poptquest02_help.html"
            },
        .
        .
        .
        ]
    }
]
```

## Choices section

This section defines choices that are associated with a question.

| Field | Description |
| :--- | :----: |
| `id` | ID of the answer. This needs to be unique between 3 and 128 characters and contain only alphanumeri and underscore (`_`) characters. |
| `name` | Name of the answer. The name can be up to `128` characters.
| `description` | The description of the choice. This text is displayed when answering the question. The description can be up to 2048 characters. |
| `resources` | Contains a list of further resources to help understand the context of the question. |
| `improvments` | Contains a list of improvment steps to remove identified risks. |

### Example

```JSON
"choices": [
    {
        "id": "choice_1,
        "title": "Option 1",
        "description": "We require that new interfaces monitor the inbound and outbound data to systems.",
        "resources": [
            {
                "description": "This is helpful text for the first choice",
                "url": "https://example.com/popt01_help.html"
            },
            .
            .
            .
        ],
        "improvements": [
            {
                "description": "This is helpful text for improvement of this choice.",
                "url": "https://example.com/popt01_iplan_01.html"
            },
           .
            .
            .
        ]
        .
        .
        .
    }
]
```

## Risks section

This section defines the risks associated with the selected choice to the questions in the pillar.

| Field | Description |
| :--- | :----: |
| `condition` | A boolean expression of the choices that maps those choices to a risk level. |
| `risk` | Indicates the risk associated with the question. Valid values are `HIGH`, `MEDIUM`, `NONE`.

### Example

```JSON
"risks": [
       {
         "condition": "choice_1 && choice_2 && choice_3",
         "risk": "NONE"  
       },
       {
         "condition": "((choice_1 || choice_2) && choice_3) || (!choice_1 && choice_3)",
         "risk": "MEDIUM"
       },
       {
         "condition": "default",
         "risk": "HIGH"
       }
]
```
